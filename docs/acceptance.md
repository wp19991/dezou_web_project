# Acceptance

## Manual Smoke Test

1. 使用默认配置启动服务，打开首页，确认能正常加载牌桌页面。
2. 以 `用户参与` 模式创建一个 session，确认：
   - 用户名默认是 `玩家`
   - 创建前可编辑
   - 创建后设置区隐藏，只剩“开始新一手”
3. 在 `用户参与` 模式下开始一手，确认：
   - 自己能看到自己的两张底牌
   - 其他玩家隐藏手牌显示为牌背，而不是空白
   - 当前手时间线不显示
4. 刷新页面或通过“载入”重新打开同一个 session，确认：
   - 仍然恢复为 `用户参与` 模式
   - 用户名保持锁定，不可修改
5. 再创建一个 `旁观模式` session，确认：
   - 动作区和聊天区隐藏
   - 当前手时间线可见
6. 在创建 session 时输入固定随机种子，连续开始两手，确认同一 session 下 hand seed 自动递增派生。
7. 结束一手牌后，确认历史列表立即可查询，且无需再开始下一手就能打开 replay。
8. 若进入摊牌，确认赢家、赢取金额、参与比牌座位和最佳牌型都有展示。
9. 若以 `POKER_BOT_NAMES` 启动服务，确认首页创建区展示的候选名称池与环境变量一致。

## Known Limits

- 仅支持单进程运行时状态。
- 前端采用轮询，不包含 WebSocket。
- 仍然没有鉴权层，`viewer_name / actor_name / speaker_name` 属于信任式接入参数。
- 内置网页上的模式限制、名称生成和人数上限主要是前端约束；直接调用后端 API 时，仍然以接口校验为准。
